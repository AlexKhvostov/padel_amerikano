# Деплой на Hostland

## Окружение

| Параметр | Значение |
|----------|----------|
| Домен | https://padel.ballaball.xyz/ |
| PHP | 8.2, mod_php (Apache) |
| MySQL | 8.0 |
| Хост БД | `mysql80.hostland.ru:3308` |

## 1. Загрузка файлов

Загрузите содержимое репозитория на хостинг (Git или SFTP).

**Рекомендуется:** указать document root домена на папку `public/`.

Если document root — корень репозитория, сработает корневой `.htaccess`.

## 2. Конфигурация `.env`

В корне проекта (рядом с `config/`) создайте `.env`:

```env
APP_URL=https://padel.ballaball.xyz
APP_ENV=production
APP_DEBUG=false
TRUSTED_PROXIES=

DB_HOST=mysql80.hostland.ru
DB_PORT=3308
DB_NAME=host1708875_padelbd
DB_USER=host1708875_upadel
DB_PASSWORD=ваш_пароль

APP_SECRET=случайная_строка_не_короче_32_символов
```

Файл `.env` не хранится в git.
`TRUSTED_PROXIES` обычно оставляется пустым. Если перед Apache используется
reverse proxy, укажите здесь только его доверенные IP через запятую — это
необходимо для корректной блокировки попыток входа по IP пользователя.

## 3. База данных

Для новой базы выполните `sql/schema.sql`.

Если старая схема уже установлена, выполните миграцию
`sql/migrations/002_round_schedule_status.sql`, а затем
`sql/migrations/003_viewer_access.sql` и
`sql/migrations/004_short_view_links.sql`, затем
`sql/migrations/005_company_soft_delete.sql` и
`sql/migrations/006_login_bruteforce_protection.sql`, затем
`sql/migrations/007_multi_tournament.sql`, затем
`sql/migrations/008_tournament_activity.sql`.

Если миграция `002` уже применялась ранее, выполните последовательно `003`, `004`, `005`, `006`, `007` и `008`.
Если `003` уже применялась — выполните последовательно `004`, `005`, `006`, `007` и `008`.
Если `004` уже применялась — выполните последовательно `005`, `006`, `007` и `008`.
Если `005` уже применялась — выполните последовательно `006`, `007` и `008`.
Если `006` уже применялась — выполните `007`, затем `008`.
Если `007` уже применялась — выполните только `008_tournament_activity.sql`.

Миграция `006` обязательна: она добавляет учёт названия компании в журнал
ошибок входа. Без неё обновлённый обработчик административного входа работать
не будет.

Перед миграцией `007` создайте резервную копию базы. Затем:

1. Выполните `007_multi_tournament.sql` в phpMyAdmin.
2. Только после успешного завершения загрузите новый PHP и frontend.
3. Не запускайте `007` повторно.

После успешной миграции `007` выполните `008_tournament_activity.sql` один раз.
Она не меняет результаты матчей, а добавляет время последнего обновления турнира.

Миграция создаёт для каждой существующей компании первый турнир и переносит
в него текущие раунды, матчи, результаты и состав. Старые зрительские ссылки
остаются действительными.

Проверка миграции:

```sql
SELECT COUNT(*) AS companies_count FROM companies;
SELECT COUNT(*) AS tournaments_count FROM tournaments;
SELECT COUNT(*) AS rounds_without_tournament FROM rounds WHERE tournament_id IS NULL;
```

Число турниров сразу после миграции должно быть не меньше числа компаний,
а `rounds_without_tournament` должно быть равно `0`.

## 4. Проверка

Откройте в браузере:

- https://padel.ballaball.xyz/ — главный экран
- https://padel.ballaball.xyz/api/health — статус API и подключения к БД

Ожидаемый ответ health:

```json
{"status":"ok","db":true,"php":"8.2.x"}
```

## 5. Права доступа

- PHP должен иметь доступ на чтение `.env` и всех файлов проекта
- Запись на диск не требуется (кроме логов хостинга)

## 6. Обязательные production-проверки

1. Включите бесплатный TLS-сертификат для домена в панели Hostland.
2. Настройте перенаправление HTTP → HTTPS в панели хостинга.
3. Убедитесь, что `APP_DEBUG=false`.
4. Вызовите защищённый API из приложения и убедитесь, что Apache передаёт
   заголовок `Authorization` в PHP (правило уже добавлено в `public/.htaccess`).
5. Не публикуйте `.env` и не размещайте корень проекта внутри document root.
6. На тестовой компании проверьте защиту входа: после трёх неверных кодов
   четвёртая попытка с того же IP должна вернуть HTTP 429 и сообщение о
   десятиминутной блокировке. Другой IP при этом не блокируется.

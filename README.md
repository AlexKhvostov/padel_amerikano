# Падел Американо

Веб-сервис для проведения любительских турниров по системе «Американо» в падел-теннисе.

**Продакшен:** https://padel.ballaball.xyz/

## Стек

| Слой | Технология |
|------|------------|
| Frontend | Vanilla JS, HTML5, CSS3 (mobile-first) |
| Backend | PHP 8.2 (mod_php) |
| База данных | MySQL 8.0 (Hostland) |

## Структура проекта

```
├── config/          # env.php, database.php
├── public/          # document root (index.php, assets, api)
├── sql/schema.sql   # схема БД
├── src/             # PHP-сервисы и auth
└── docs/DEPLOY.md   # инструкция деплоя
```

## Быстрый старт (локально / сервер)

1. Скопируйте `config/.env.example` → `.env`, заполните реквизиты
2. Для новой базы выполните `sql/schema.sql`. Для существующей базы последовательно примените миграции `002` → `003` → `004` → `005` → `006` → `007` → `008`
3. Настройте Apache document root на `public/`
4. Откройте `/api/health` для проверки

Она открывает хаб компании с текущим и завершёнными турнирами.

## Модель данных

- Компания — постоянная группа с общим списком участников и историей игр.
- В компании можно создавать несколько турниров; одновременно активен только один.
- Каждый турнир имеет собственный состав из 4–36 участников, корты, раунды и рейтинг.
- Список участников одновременно является рейтингом компании: после 5 матчей сортировка идёт по доле набранных очков, средней разнице и проценту побед; меньшая выборка помечается как предварительная.
- Нажатие на участника открывает подробную статистику и историю его турниров.
- Основное меню компании остаётся неизменным, а внутри турнира над активной кнопкой «Турниры» появляется компактная панель второго уровня «Раунды / Рейтинг».
- Миграция `007_multi_tournament.sql` превращает прежние данные каждой компании в её первый сохранённый турнир без удаления матчей и результатов.
- Миграция `008_tournament_activity.sql` добавляет время изменения турнира для информационных статусов: «Идёт» до 15 минут без обновлений, «Заброшен» после 15 минут и «Собирается» для свежего черновика в течение часа. Статусы не останавливают турнир автоматически.

## Безопасность

- Названия активных компаний уникальны без учёта регистра. После логического удаления название освобождается.
- Код администратора хранится только как bcrypt-хеш.
- После 3 ошибок для сочетания IP и компании вход блокируется на 10 минут.
- После 20 ошибок с одного IP по разным компаниям вход с этого IP блокируется на 10 минут.
- В production обязательны HTTPS, `APP_DEBUG=false` и `APP_SECRET` длиной не менее 32 символов.
- Если сервис работает за reverse proxy, его IP необходимо добавить в `TRUSTED_PROXIES`; иначе значение оставляют пустым.

ТЗ: Веб-сервис «Падел Американо»
1. Общее описание

Веб-сервис для проведения любительских турниров по системе «Американо» в падел-теннисе.

Американо — турнирный формат, где:

Игроки постоянно меняют партнёров и соперников после каждого раунда
Очки считаются индивидуально для каждого игрока
Победитель определяется по наибольшей сумме личных очков

Ключевые требования:

Mobile-first интерфейс (приоритет экранов 360–430px)
Vanilla JS, без фреймворков
Бэкенд с REST API, MySQL база данных на сервере заказчика
Интерфейс полностью на русском языке
2. Технический стек

3. Система «Компаний»
3.1 Концепция

«Компания» — это изолированное пространство одного турнира или серии турниров. Каждая компания имеет своё название и простой пароль для входа. Любой, кто знает название и пароль, получает полный доступ к управлению.

3.2 Создание компании
Главный экран содержит кнопку «Создать компанию»
Пользователь вводит название компании (например: «Клуб Ракетка», «Турнир у Димы»)
Система автоматически генерирует простой цифровой пароль (4–6 цифр)
Компания сохраняется в БД
После создания на экране крупно отображается:
Компания: Клуб Ракетка
Пароль: 4872

Эти данные можно скопировать или поделиться (кнопка «Поделиться»)
3.3 Вход в компанию
На главном экране — поле поиска по названию компании
После выбора компании — поле ввода пароля
После успешного входа — переход в интерфейс управления турниром
Сессия сохраняется в localStorage: при перезагрузке страницы пользователь остаётся авторизованным
3.4 Выход
Кнопка «Выйти из компании» в настройках
Очищает localStorage, возвращает на главный экран
4. Настройки турнира

Отдельный экран «⚙️ Настройки» доступен из нижнего меню.

4.1 Параметры
Параметр	Тип	Описание
Счёт до	Радиокнопки	Выбор: до 16 или до 24 очков
Дополнительный розыгрыш (+1)	Чекбокс	При равном счёте (8:8 или 12:12) играется дополнительный розыгрыш
+1 всегда обязательный	Чекбокс	Если отмечено — дополнительный розыгрыш играется всегда, даже если счёт не равный
Количество кортов	Число	Количество одновременно задействованных кортов
4.2 Логика счёта
При игре до 16: максимум 16 очков на команду. При счёте 8:8 — играется +1 розыгрыш (итог 9:8 или 8:9)
При игре до 24: максимум 24 очка на команду. При счёте 12:12 — играется +1 розыгрыш (итог 13:12 или 12:13)
Если включён флаг «+1 всегда»: дополнительный розыгрыш разыгрывается после каждого матча, даже при неравном счёте
В таком случае максимальный счёт матча: 17 (при игре до 16) или 25 (при игре до 24)
4.3 Изменение настроек
Настройки можно менять до начала первого раунда
После начала турнира — настройки заблокированы с предупреждением: «Турнир начат. Изменение настроек недоступно»
Кнопка «Сбросить турнир» сбрасывает все раунды и разблокирует настройки (с подтверждением)
5. Список игроков
5.1 Добавление игрока

Форма добавления содержит поля:

Имя (обязательное) — текстовое поле
Telegram (необязательное) — ник вида @username или полная ссылка https://t.me/username
5.2 Отображение списка

Каждый игрок отображается карточкой:

[Аватар-инициалы]  Иван Петров
                   @ivanpetrov  (кликабельная ссылка)
                   [✏️ Редактировать]  [🗑️ Удалить]

Telegram отображается как кликабельная ссылка, открывающая Telegram
Если ник указан без @ — добавлять автоматически
Если указана полная ссылка — отображать как @username
5.3 Ограничения
Минимум 4 игрока для старта турнира
Максимум 36 игроков
Нельзя добавить двух игроков с одинаковым именем в одной компании
Если игрок удаляется после начала турнира — его исторические результаты сохраняются, но он помечается как «неактивный»
5.4 Счётчик

В верхней части экрана показывается: «Игроков: 12 / 36»

6. Генерация раундов
6.1 Алгоритм ротации
Раунды генерируются автоматически по случайному алгоритму
В каждом раунде игроки разбиваются на случайные пары
Пары случайно распределяются по кортам: 1 корт = 1 матч = 2 пары (4 игрока)
Система стремится к тому, чтобы:
Каждый игрок как можно реже играл с одним и тем же партнёром
Каждый игрок как можно реже встречался с одними и теми же соперниками
6.2 Нечётное количество игроков (не кратно 4)
Количество игроков	Поведение
Кратно 4 (8, 12, 16...)	Все играют в каждом раунде
Остаток 2 (10, 14, 18...)	2 игрока пропускают раунд («скамейка»), чередуются по очереди
Остаток 3 (например, 7)	Нестандартная ситуация — предупредить пользователя. Рекомендовать добавить/убрать игрока

Игроки на «скамейке» чётко отображаются в раунде: «Пропускают раунд: Антон, Мария»

6.3 Количество раундов
Рекомендуемое количество раундов рассчитывается автоматически:
Формула: N игроков / (4 × кол-во кортов) × коэффициент ротации
Организатор может вручную добавлять новые раунды кнопкой «+ Добавить раунд»
Раунды нумеруются по порядку: Раунд 1, Раунд 2...
7. Экран «Раунды»
7.1 Структура экрана
Список раундов в виде аккордеона или табов
Текущий активный раунд раскрыт по умолчанию
Завершённые раунды можно свернуть
7.2 Отображение матча
┌─────────────────────────────────────────┐
│  КОРТ 1                                 │
│                                         │
│  🟦 Иван + Сергей                       │
│  vs                                     │
│  🟥 Олег + Дмитрий                      │
│                                         │
│  [ 8 ]  :  [ 8 ]   ← поля ввода счёта  │
│                                         │
│  [     Сохранить     ]                  │
└─────────────────────────────────────────┘

7.3 Ввод счёта
Два числовых поля — одно для каждой команды
Валидация при сохранении:
Сумма очков ≤ максимальный счёт + 1
Оба поля заполнены
Разница между командами ≥ 1 (ничьей не бывает, кроме случая до +1)
При корректном вводе — матч помечается зелёной галочкой ✅
Уже сохранённый счёт можно редактировать (кнопка «✏️»)
7.4 Завершение раунда
Когда все матчи раунда завершены — появляется кнопка «▶ Следующий раунд»
При нажатии генерируется следующий раунд с новой ротацией
Генерация учитывает историю предыдущих пар
8. Экран «Рейтинг»
8.1 Таблица рейтинга
#	Игрок	Telegram	Матчей	Очков	Побед	Поражений
🥇 1	Иван	@ivan	6	94	5	1
🥈 2	Сергей	@sergey	6	88	4	2
🥉 3	Олег	—	6	81	3	3
8.2 Правила подсчёта
Очки — сумма всех очков игрока за все матчи (очки своей команды в каждом матче)
Победа — команда игрока набрала больше очков в матче
Поражение — команда игрока набрала меньше очков в матче
8.3 Сортировка
По очкам (по убыванию)
При равных очках — по победам (по убыванию)
При равных очках и победах — по алфавиту
8.4 Обновление
Рейтинг обновляется в реальном времени после каждого сохранённого счёта
Каждый игрок видит своё текущее место
9. Структура базы данных (MySQL)
-- Компании
CREATE TABLE companies (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    password    VARCHAR(10) NOT NULL,
    settings    JSON,           -- настройки турнира
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Игроки
CREATE TABLE players (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    company_id  INT NOT NULL,
    name        VARCHAR(100) NOT NULL,
    telegram    VARCHAR(100),
    is_active   TINYINT DEFAULT 1,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (company_id) REFERENCES companies(id)
);

-- Раунды
CREATE TABLE rounds (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    company_id   INT NOT NULL,
    round_number INT NOT NULL,
    created_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (company_id) REFERENCES companies(id)
);

-- Матчи
CREATE TABLE matches (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    round_id     INT NOT NULL,
    court_number INT NOT NULL,
    FOREIGN KEY (round_id) REFERENCES rounds(id)
);

-- Состав команд в матче
CREATE TABLE match_players (
    id         INT AUTO_INCREMENT PRIMARY KEY,
    match_id   INT NOT NULL,
    player_id  INT NOT NULL,
    team       TINYINT NOT NULL,   -- 1 или 2
    FOREIGN KEY (match_id) REFERENCES matches(id),
    FOREIGN KEY (player_id) REFERENCES players(id)
);

-- Счёт матча
CREATE TABLE match_scores (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    match_id     INT NOT NULL UNIQUE,
    score_team1  INT,
    score_team2  INT,
    is_finished  TINYINT DEFAULT 0,
    updated_at   DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (match_id) REFERENCES matches(id)
);

12. UX / UI требования
12.1 Mobile-first
Основной целевой экран: 360px – 430px ширины
Все кнопки и поля ввода — не менее 48px высотой (для удобного нажатия пальцем)
Числовые поля счёта открывают цифровую клавиатуру (inputmode="numeric")
Нет горизонтального скролла
12.2 Общие принципы
Чёткая визуальная иерархия: название матча, команды, поля счёта
Цветовое различие команд (синяя / красная метка)
Завершённые матчи и раунды визуально отличаются от активных
Все деструктивные действия (удалить, сбросить) требуют подтверждения
12.3 Состояния загрузки
При каждом запросе к API — индикатор загрузки (спиннер)
При ошибке сети — понятное сообщение пользователю с кнопкой «Повторить»
12.4 Уведомления
Toast-уведомления (всплывающие снизу) при:
Успешном сохранении счёта
Добавлении игрока
Ошибке валидации
13. Безопасность
Пароль компании хранится в БД в хешированном виде (bcrypt или MD5 минимум)
Все запросы к API проверяют принадлежность данных к компании (по session token или JWT)
Защита от SQL-инъекций — использовать параметризованные запросы (PDO / prepared statements)
Базовая защита от брутфорса: не более 10 попыток входа с одного IP за 10 минут
14. Дополнительные пожелания (Nice to have)
Кнопка «Поделиться» — копирует название и пароль компании для отправки в мессенджер
Экспорт результатов — скачать итоговый рейтинг в виде текста или изображения
Поддержка PWA (Progressive Web App) — добавление на домашний экран телефона
